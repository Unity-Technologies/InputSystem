◆ AllocateDeviceId()

int ISX.LowLevel.IInputRuntime.AllocateDeviceId ( )

Allocate a new unique device ID.

Returns
A numeric device ID that is not InputDevice.kInvalidDeviceId.

Device IDs are managed by the runtime. This method allows creating devices that can use the same ID system but are not known to the underlying runtime.

Queue an input event.

This method has to be thread-safe.

Parameters
ptrPointer to the event data. Uses the InputEvent format.

◆ Update()

void ISX.LowLevel.IInputRuntime.Update ( InputUpdateType  type)

Manually trigger an update.

Parameters
typeType of update to run. If this is a combination of updates, each flag that is set in the mask will run a separate update.

Updates will flush out events and trigger onBeforeUpdate and onUpdate. Also, newly discovered devices will be reported by an update is run.

◆ onDeviceDiscovered

Action<int, string> ISX.LowLevel.IInputRuntime.onDeviceDiscovered
set

Set delegate to be called when a new device is discovered.

The runtime should delay reporting of already present devices until the delegate has been put in place and then call the delegate for every device already in the system.

First parameter is the ID assigned to the device, second parameter is a description in JSON format of the device (see InputDeviceDescription.FromJson).
